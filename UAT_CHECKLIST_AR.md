# سيناريوهات اختبار UAT الشاملة قبل التسليم

هذا الملف يحتوي سيناريوهات تشغيل فعلية (End-to-End) للتأكد أن النظام بالكامل يعمل بشكل صحيح ومترابط بين الأقسام.

## تعليمات التنفيذ السريع

- [ ] نفّذ السيناريوهات بالترتيب من 1 إلى 14.
- [ ] سجّل لكل سيناريو: `Pass` أو `Fail`.
- [ ] عند فشل أي سيناريو: وثّق الخطوة التي فشلت + صورة شاشة + رسالة الخطأ.
- [ ] أعد اختبار السيناريو بعد الإصلاح مباشرة.

---

## 1) RBAC-01 | إدارة الموظفين (Owner فقط)

**الدور:** مالك + مدير مبيعات  
**الخطوات:**
- [ ] سجّل دخول كـ مالك وأنشئ موظفًا جديدًا.
- [ ] سجّل دخول كـ مدير مبيعات وحاول إنشاء موظف.
**النتيجة المتوقعة:**
- [ ] إنشاء الموظف ينجح مع المالك فقط.
- [ ] مدير المبيعات لا يستطيع الإضافة/الحذف.

## 2) RBAC-02 | ظهور التبويبات حسب الدور

**الدور:** مالك / مدير مبيعات / مندوب / محاسب / مدير إنتاج  
**الخطوات:**
- [ ] سجّل دخول بكل دور على حدة.
- [ ] راجع التبويبات الظاهرة في الشريط الرئيسي.
**النتيجة المتوقعة:**
- [ ] كل دور يرى فقط التبويبات الخاصة به.
- [ ] تبويب SEO يظهر للمالك فقط.

## 3) LEADS-01 | استيراد ليدز من القنوات

**الدور:** مالك  
**الخطوات:**
- [ ] شغّل مزامنة ليدز من Facebook.
- [ ] كرر مع Google / LinkedIn / Email.
**النتيجة المتوقعة:**
- [ ] الليدز تُنشأ بحالة `جديد`.
- [ ] يتم تحويلها تلقائيًا لمدير المبيعات (عند تفعيل auto-route).
- [ ] يظهر حدث الاستيراد في السجل.

## 4) LEADS-02 | توزيع الليدز على المندوبين

**الدور:** مدير مبيعات  
**الخطوات:**
- [ ] افتح قائمة الليدز الجديدة.
- [ ] عيّن 3 ليدز على مندوبين مختلفين.
- [ ] ألغِ تعيين ليد واحد.
**النتيجة المتوقعة:**
- [ ] التعيين يحدث فورًا.
- [ ] يظهر في Timeline: `تعيين المندوب` أو `إلغاء التعيين`.
- [ ] الليد يظهر في لوحة المندوب المعيّن له فقط.

## 5) LEADS-03 | إجراءات المندوب على ليد مخصص له

**الدور:** مندوب  
**الخطوات:**
- [ ] افتح ليد معيّن لك.
- [ ] سجّل تفاعل (مكالمة/شات) مع ملاحظة.
- [ ] أضف دليل (رابط تسجيل/محادثة) ومدة.
- [ ] عيّن Follow-up.
**النتيجة المتوقعة:**
- [ ] التفاعل يتسجل في Timeline.
- [ ] الدليل يظهر في أرشيف العميل.
- [ ] التفاعل الذي يحتوي دليل يدخل `QA pending`.

## 6) LEADS-04 | منع المندوب من التعديل على ليد ليس له

**الدور:** مندوب  
**الخطوات:**
- [ ] حاول تعديل حالة/متابعة ليد غير معيّن لك.
**النتيجة المتوقعة:**
- [ ] العملية تُرفض ولا يتم حفظ أي تغيير.

## 7) LEADS-05 | إغلاق خسارة بجودة إلزامية

**الدور:** مندوب  
**الخطوات:**
- [ ] حاول إغلاق ليد كـ `مغلق - خسارة` بدون سبب أو بدون Quality Gate.
- [ ] أعد المحاولة مع سبب + Quality Gate + ملاحظة.
**النتيجة المتوقعة:**
- [ ] الإغلاق يُرفض في الحالة الأولى.
- [ ] ينجح في الحالة الثانية مع تسجيل السبب في Timeline.

## 8) SALES-QUOTE-01 | دورة عرض السعر حتى الفاتورة

**الدور:** مندوب/مدير مبيعات -> مالك -> محاسب  
**الخطوات:**
- [ ] أنشئ عرض سعر من ليد نشط.
- [ ] اعتمد العرض كـ مالك.
- [ ] افتح شاشة المحاسب.
**النتيجة المتوقعة:**
- [ ] العرض يبدأ `قيد اعتماد المالك`.
- [ ] بعد الاعتماد يتحول `معتمد`.
- [ ] يتم إنشاء فاتورة تلقائيًا مرتبطة بالعرض.
- [ ] العميل يظهر للمحاسب ضمن المسار المالي المعتمد.

## 9) FIN-01 | المصروفات ومسار الاعتماد

**الدور:** مدير إنتاج/محاسب -> مالك  
**الخطوات:**
- [ ] أنشئ مصروف جديد.
- [ ] راقبه في قائمة الاعتمادات.
- [ ] اعتمد المصروف كـ مالك.
**النتيجة المتوقعة:**
- [ ] المصروف يبدأ `قيد الاعتماد`.
- [ ] لا يتحول `مدفوع` قبل الاعتماد.
- [ ] بعد الاعتماد يسمح بالتنفيذ المالي.

## 10) FIN-02 | القيود اليومية وتكويد الحسابات

**الدور:** محاسب  
**الخطوات:**
- [ ] أدخل قيدًا يدويًا متوازنًا (مدين = دائن) بأكواد صحيحة.
- [ ] أدخل قيدًا غير متوازن.
- [ ] أدخل قيدًا بكود حساب غير موجود بالدليل.
**النتيجة المتوقعة:**
- [ ] القيد المتوازن بالأكواد الصحيحة ينجح.
- [ ] القيد غير المتوازن يُرفض.
- [ ] القيد بكود غير موجود يُرفض.

## 11) FIN-03 | إقفال الشهر وإعادة الفتح

**الدور:** مالك + محاسب  
**الخطوات:**
- [ ] أقفل شهرًا ماليًا.
- [ ] حاول إضافة فاتورة/مصروف/قيد داخل نفس الشهر.
- [ ] قدّم طلب إعادة فتح من المحاسب.
- [ ] اعتمد الطلب من المالك.
**النتيجة المتوقعة:**
- [ ] الإدخالات المالية تُمنع أثناء الإقفال.
- [ ] إعادة الفتح لا تتم مباشرة من المحاسب.
- [ ] بعد اعتماد المالك تعود العمليات للعمل.

## 12) OPS-01 | حجوزات الاجتماعات/التصوير/المعدات

**الدور:** مندوب/مدير إنتاج -> مالك -> محاسب  
**الخطوات:**
- [ ] أنشئ اجتماع خارجي بتكلفة.
- [ ] اعتمده كـ مالك.
- [ ] نفّذ المطالبة كـ محاسب.
**النتيجة المتوقعة:**
- [ ] الحالة المالية: `بانتظار اعتماد مالك` -> `بانتظار تنفيذ محاسب` -> `منفذ`.
- [ ] يتولد مصروف تلقائي مرتبط بطلب الحجز.

## 13) OPS-02 | دورة عهدة الإنتاج كاملة

**الدور:** مدير إنتاج -> مالك -> محاسب -> مدير إنتاج  
**الخطوات:**
- [ ] أنشئ طلب عهدة.
- [ ] اعتماد المالك.
- [ ] صرف المحاسب.
- [ ] استلام مدير الإنتاج.
- [ ] إضافة بنود الصرف ثم إرسال التسوية.
- [ ] اعتماد/إقفال التسوية.
**النتيجة المتوقعة:**
- [ ] الدورة تمر بكل الحالات بدون كسر.
- [ ] القيود/الحركات المالية المرتبطة تُسجّل.

## 14) NOTIFY-01 | نظام التنبيهات الكامل

**الدور:** جميع الأدوار  
**الخطوات:**
- [ ] أنشئ حالات تولد تنبيهات (ليد غير موزع، متابعة متأخرة، مطالبات إنتاج، عروض سعر تنتظر اعتماد).
- [ ] راجع مركز التنبيهات بكل دور.
- [ ] اضغط على تنبيه للانتقال.
**النتيجة المتوقعة:**
- [ ] التنبيهات تُفلتر حسب `role` و`targetUserId`.
- [ ] المندوب يرى تنبيهاته الشخصية فقط (مثل متابعاته المتأخرة).
- [ ] التنبيه يفتح التبويب الصحيح ضمن صلاحيات المستخدم.
