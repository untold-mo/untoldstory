# خطة ربط الواجهة بالباك اند — ما تم واللي جاي

## تم تنفيذه في الكود

### 1) المستخدمون في PostgreSQL
- **`GET/POST /api/users`**, **`PATCH/DELETE /api/users/:id`**
- **`/auth/login`** و **`/auth/me`**

### 2) العملاء اليدويّون
- **`/api/manual-customers`**

### 3) الفواتير والمصروفات
- **`/api/invoices`**, **`/api/expenses`**

### 4) عروض الأسعار والقيود وسياسة المحاسبة
- **`/api/price-quotes`**, **`/api/manual-journals`**, **`GET/PATCH /api/accounting-policy`**

### 5) أشهر الإغلاق والأهداف والتدقيق والعهدة والحجوزات (جديد)
- **`GET/POST /api/closed-months`** — `POST /close` و `POST /reopen` (مالك)
- **`GET/PATCH /api/monthly-targets/:repId`** (مالك / مدير مبيعات)
- **`GET/PATCH /api/custody-settings`** — خريطة فئة المصروف → حساب (محاسب للتعديل)
- **`GET/POST /api/audit-events`**
- **`GET/POST/PUT /api/custody-funds`**
- **`GET/POST/PATCH /api/shoot-bookings`**, **`equipment-bookings`**, **`meeting-bookings`**

### 6) حالة مساحة العمل والحضور
- **`GET/PATCH /api/workspace-state`** — دمج جزئي لـ `doc_json` (شجرة الحسابات، السنوات المالية المقفلة، الأرصدة الافتتاحية، إعدادات الطباعة والتكاملات، سير العمل، SLA، جودة الليدز، المعدات، اعتمادات الرواتب وطلباتها، طلبات إعادة فتح الفترة، **أكواد اليومية** `journalCodebook`، **دفتر تكويد المصروفات** `expenseCodebook`، **بادئة كود العميل** `customerCodePrefix`، **عروض فلترة المصروفات** `expenseSavedViews`، **يوم إرسال الرواتب التلقائي** `payrollAutoSendDay` (محاسب/مالك)، **تعليقات الكيانات** `entityComments` (مالك)، **تصعيد مصروفات** `expenseEscalations` (محاسب/مالك)، **وضع الواجهة** `uiVisualMode` (مالك)، **المهام الشخصية** `personalTodosByUserId` (كل مستخدم يحدّث مفتاحه فقط؛ المالك يمكنه الكل)، إلخ) مع صلاحيات حسب الدور على المفاتيح العليا.
- **`GET/POST /api/attendance-records`** — سجلات الحضور/الانصراف (محاسب / مالك).

مع **`VITE_DATA_SOURCE=server`** تُحمّل هذه الكيانات من الـ API وتُحدَّث عبره (بدون حفظها في `localStorage` حيث ينطبق ذلك).

- **قيود اليومية اليدوية (إدخال من المحاسب/المالك):** **`POST /api/manual-journals`** كما في التحصيل والعهدة؛ **`DELETE /api/manual-journals/:id`** لحذف قيد (مع قيود: شهر/سنة مغلقة، أو ارتباط بتحصيل فاتورة/عهدة).
- **معدات الحجز الرئيسية:** إضافة من واجهة «الإعدادات» أو «الحجوزات» مع **`PATCH /api/workspace-state`** (`equipmentItems`) لمالك / محاسب / مدير إنتاج.

## الهجرة

من **`api`**:

```bash
npm run db:migrate
```

هجرات ذات صلة: **`20260510120000_closed_targets_audit_custody_bookings`**، **`20260510140000_workspace_state_attendance`**

إن لزم: **`دليل-ربط-قاعدة-البيانات.md`** (P3005 / EPERM).

---

## تشغيل سريع

1. **`api/.env`**: `DATABASE_URL` + `DIRECT_URL`.
2. **`.env.local`**: `VITE_API_URL` + `VITE_DATA_SOURCE=server`.
3. `cd api && npm run dev` ثم تشغيل الواجهة.

---

## ما زال محلياً أو مختلطاً (يمكن لاحقاً)

- سلوك متزامن أقوى عند فشل الشبكة (إعادة محاولة/تعارض) لبعض عمليات الـ PATCH.
- إعدادات أخرى ما زالت على `localStorage` فقط إن وُجدت خارج هذا المسار (مثلاً يوم إرسال الرواتب التلقائي من الواجهة).
