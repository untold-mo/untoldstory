  نشر التطبيق على Hostinger — ما تم تنفيذه على الجهاز
═══════════════════════════════════════════════════════════════

تم تنفيذ محلياً:
  • npm ci
  • npm run build
  • نسخ ملفات التطبيق فقط (بدون مجلدات إضافية داخل dist) إلى:
      hostinger-dist\
  • ضغط الملفات في:
      hostinger-upload.zip

محتوى الرفع الصحيح للموقع:
  • index.html
  • مجلد assets\  (يحتوي ملفات .js و .css)

ملاحظة: مجلد dist يحتوي عندك ملفات أخرى (مثل globaluntoldstory).
  الرفع على Hostinger يجب أن يكون من hostinger-dist أو من محتوى hostinger-upload.zip
  وليس نسخ مجلد dist بالكامل إن كان فيه مواقع أخرى.

═══════════════════════════════════════════════════════════════
  الخطوات التي لا يمكن تنفيذها إلا من حسابك على Hostinger
═══════════════════════════════════════════════════════════════

1) تسجيل الدخول إلى hPanel (Hostinger).

2) فتح File Manager → الدخول إلى المجلد:
     public_html
   (أو مجلد الدومين إذا كان الموقع على دومين فرعي/دومين إضافي).

3) رفع الملفات:
   أ) إما فك ضغط hostinger-upload.zip داخل public_html
      بحيث يصبح:
        public_html/index.html
        public_html/assets/...
   ب) أو رفع محتويات مجلد hostinger-dist يدوياً بنفس الشكل.

4) التأكد من عدم وجود index قديم يتعارض:
   • إذا كان فيه index.htm أو index.php للصفحة الافتراضية القديمة،
     احذفه أو أعد تسميته بعد أخذ نسخة احتياطية،
     لكي يُفتح index.html الخاص بالتطبيق.

5) تفعيل SSL:
   من hPanel → SSL → تفعيل شهادة مجانية للدومين.

6) التحقق من الدومين:
   • إن كان الدومين جديداً، تأكد أن الـ DNS يشير لسيرفر Hostinger
     (سجلات A أو استخدام أسماء الـ nameservers التي يعطيك إياها Hostinger).

7) فتح الموقع في المتصفح واختبار:
   • إذا الصفحة بيضاء: افتح F12 → Network وتأكد أن ملفات assets/*.js تُحمّل (ليست 404).

8) SPA وملف .htaccess:
   • تمت إضافة public/.htaccess ليُنسَخ تلقائياً ضمن مجلد dist عند npm run build
     (Rewrite إلى index.html). ارفعه مع index.html إلى public_html.
   • إذا كان الموقع داخل مسار وليس الجذر (مثل public_html/myapp/) غيّر داخل .htaccess
     RewriteBase والمسار في السطر الأخير ليطابق ذلك المجلد قبل الرفع؛
     وبنفس الوقت عيّن في .env.production القيمة:
       VITE_BASE_PATH=/myapp/
     ثم أعد npm run build.

9) متغيرات البيئة للدومين الجديد (اختيارية — قبل npm run build):
   • أنشئ ملف .env.production من النموذج .env.production.example
   • VITE_PUBLIC_APP_URL=https://YOUR-DOMAIN.TLD
       (HTTPS بدون سلاش أخيرة) — للروابط الموحّدة ومسارات OAuth عند CDN/بروكسي.
   • VITE_OAUTH_API_ORIGIN=https://api.YOUR-DOMAIN.TLD
       فقط إذا كان خادم /api مختلفاً عن عنوان الواجهة.
   • خادم Node التجريبي backend/oauth-server.mjs يقرأ APP_ORIGIN — راجع backend/.env.example
